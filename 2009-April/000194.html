The library is certainly in a subfolder called Rlib - this is exactly 
where I want it, and where I have my mbe library. Using the full path 
gives exactly the same message. The problem is not that R can't find the 
library, if I give an entirely wrong path I get a different message. 
Strangely however, this message depends on which wrong path I give. For 
example:
library(COSTcore,lib.loc="/nr/user/david")

gives

Error in library(COSTcore, lib.loc = "/nr/user/david") :
  there is no package called 'COSTcore'

Which is fair enough. This however:

library(COSTcore,lib.loc="/nr/user/david/COST")

gives

Error: invalid version specification

Which is a bit odd, and this:

library(COSTcore,lib.loc="/nr/user/david/COST/Rlib")

gives

-------------------------------------------------------------------
Error in gzfile(file, "r") : unable to open connection
In addition: Warning message:
In gzfile(file, "r") : cannot open compressed file ''
Error : .onLoad failed in 'loadNamespace' for 'COSTcore'
Error: package/namespace load failed for 'COSTcore'


Which is where we started.

Not using the most recent version of R is clearly a problem, and I'll 
try to get the most recent version installed. It isn't a problem 
immediately, because we are going to make a Windows version which will 
use the most recent version (hopefully), but it will be soon.

> You are not using a full path so COSTcore is likely to be on a "Rlib" 
> folder under the folder where you run the install command.
>
> Try something like
>
> R CMD INSTALL -l ~/Rlib COSTcore_1.2-05.tar.gz
>
> and load the library with
>
> library(COSTcore,lib.loc="~/Rlib")
>
> or else use a full path to where you want to install.
>
> A more important problem. We've decided to develop for the latest 
> version of R, if your library will be written in R261 it will not run on 
> the latest versions (there were some important changes since) which 
> means that to use your library users will have to have installed on 
> their computers two R versions and keep in 261 the old versions of COST 
> packages. It will be a mess !!
>
> Besides you will not be able to use the latest enhancements of COSTcore 
> and you maybe using an old data structure.
