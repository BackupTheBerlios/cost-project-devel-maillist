Hi all,

Good work Mathieu.
I installed the library from the zip file and the examples in the help files ran successfully.

MATURITY
Something for development -  Some maturity scales are character format not 1 to n and I think there are a few species, e.g. herring, in which Stages 1 & 2 are immature rather than just 1 so the current version of bpEstim code below won't work for these exceptions. 

I think it's an general issue and we either need to 
i) only accept Maturity scales with immature as 1
ii) as part of validation convert the different Maturity scales to a common coding, at or least numeric with immature as 1
iii) have functions that can deal with the matScale field and maybe have a immatureCodes argument, e.g. immatureCodes=c("I", 1)


SPECIES
Is my understanding of the functions correct? - the species argument allows a subset of the data to be used but the functions do not use species as a strata. So the correct way to use the functions for several species is to specify one at a time. Specifying more than one species e.g., species = c("Gadus morhua", "Solea solea") will select cod and sole then give one output using the data for both. 

DATASETS
Another step in testing the functions is to try some more datasets. I've lost track of which datasets have been altered to meet the latest format and the COSTdata folder on CVS is empty. Please can someone update me on this? 

Thanks, David
